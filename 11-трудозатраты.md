# Трудозатраты

Ресурс: `/api/taskexpenses`

Трудозатраты позволяют учитывать время и стоимость работ, выполненных по заявке. Они могут списываться на исполнителей и использоваться для анализа эффективности.

## Метод GET. Получение трудозатрат по заявке и определенных трудозатрат

### Поля трудозатрат для получения

| Название | Тип | Описание |
| :--- | :--- | :--- |
| `Id` | Int | Идентификатор трудозатраты |
| `UserId` | Int | Идентификатор исполнителя, за которого списана трудозатрата |
| `UserName` | String | Имя исполнителя, за которого списана трудозатрата |
| `Date` | DateTime | Дата списания трудозатрат |
| `Minutes` | Int | Количество минут |
| `Rate` | Decimal | Рейт |
| `Comments` | String | Комментарий |
| `TaskId` | Int | Номер заявки, к которой относятся трудозатраты |
| `EditorId` | Int | Идентификатор пользователя, создавшего или изменившего последним трудозатрату |
| `EditorName` | String | Имя пользователя, создавшего или изменившего последним трудозатрату |

### Получение всех трудозатрат для заявки

**Вызов:**
```
GET /api/taskexpenses?taskid={taskid}&fields={fieldList}&pagesize={value}&page={value}
```

#### Возможные параметры

| Параметр | Обязательность | Описание |
| :--- | :---: | :--- |
| `taskid` | Да | Номер заявки, трудозатраты которой вы хотите получить |
| `fields` | Нет | Список полей, которые вы хотите получить |
| `page`, `pagesize` | Нет | Параметры отображения таблицы |

#### Примеры

**Пример: Получить идентификатор, минуты и дату для заявки № 143199**
```
GET /api/taskexpenses/?taskid=143199&fields=Id,Minutes,Date
```

#### Пример вывода

```
GET /api/taskexpenses/?taskid=243
```

```xml
<TaskExpensesList>
  <Expenses>
    <Expense>
      <Id>53</Id>
      <UserName>Администратор</UserName>
      <UserId>43</UserId>
      <Date>02.09.2015 0:00:00</Date>
      <!-- ... другие поля ... -->
    </Expense>
    <Expense>
      <Id>54</Id>
      <UserName>Админ 2</UserName>
      <UserId>70</UserId>
      <Date>02.09.2015 0:00:00</Date>
      <!-- ... другие поля ... -->
    </Expense>
  </Expenses>
  <Paginator>
    <Count>2</Count>
    <Page>1</Page>
    <PageCount>1</PageCount>
    <PageSize>25</PageSize>
    <CountOnPage>2</CountOnPage>
  </Paginator>
</TaskExpensesList>
```

### Проверка доступа

- На сервисе, в котором находится заявка, у пользователя должна быть роль, у которой в типе заявки для поля «Трудозатраты» выставлено полномочие **«Просмотр»** или **«Изменение»**. Иначе вернет ошибку доступа.
- Если у сервисной роли нет прав на **«Просмотр стоимости работ»**, то поле `Rate` не будет присутствовать в ответе сервера.

### Получение определенных трудозатрат

**Вызов:**
```
GET /api/taskexpenses/{id}
```

Где `{id}` – идентификатор трудозатраты.

#### Пример вывода

```
GET /api/taskexpenses/53
```

```xml
<Expense>
  <Id>53</Id>
  <UserId>43</UserId>
  <UserName>Администратор</UserName>
  <Date>02.09.2015 0:00:00</Date>
  <Minutes>63</Minutes>
  <Rate>11,000</Rate>
  <Comments/>
  <TaskId>243</TaskId>
  <EditorId>43</EditorId>
  <EditorName>Администратор</EditorName>
</Expense>
```

### Проверка доступа

Аналогично, как и для получения всех трудозатрат по заявке.

## Метод POST. Создание трудозатрат

### Поля трудозатрат для создания

| Поле | Тип значения | Обязательность | Комментарий |
| :--- | :--- | :---: | :--- |
| `TaskId` | int | Да | Номер заявки, к которой добавляются трудозатраты |
| `Minutes` | int | Да | Количество минут |
| `UserId` | int | Нет | Идентификатор пользователя, за которого списываются трудозатраты. Если не указано – берется текущий пользователь |
| `Rate` | decimal | Нет | Рейты. Если не заданы, возьмется значение по умолчанию для роли на сервисе |
| `Comments` | string | Нет | Комментарий |
| `Date` | DateTime | Нет | Дата трудозатрат (без времени). Если не указано – берется текущая дата |

### Вызов и пример использования

**Вызов:**
```
POST /api/taskexpenses
```

#### Пример

Добавим трудозатраты в заявку № 777 — 45 минут с комментарием:

```json
{
  "TaskId": 777,
  "Minutes": 45,
  "Comments": "Добавление трудозатраты в заявку 777"
}
```

### Проверка доступа

1. Текущий пользователь должен быть назначен на сервис, в котором находится заявка `TaskId`.
2. У пользователя в типе заявки для `TaskId` в «Основных полях заявки», поле «Трудозатраты», должно стоять полномочие **«Изменение»**.
3. Если в запросе передается `UserId`, отличный от `Id` текущего пользователя, то у текущего пользователя должны быть права на **«Редактирование чужих трудозатрат»** — иначе будет ошибка.
4. Если `UserId` не является исполнителем на заявке, списать трудозатраты не получится.
5. Если у текущего пользователя нет полномочия **«Редактировать стоимость работ»**, а в запросе передается значение `Rate`, то возьмется значение `Rate` по умолчанию для роли пользователя на сервисе.

## Метод PUT. Изменение трудозатрат

### Поля трудозатрат для изменения

| Поле | Тип значения | Комментарий |
| :--- | :--- | :--- |
| `Minutes` | int | Количество минут |
| `UserId` | int | Идентификатор пользователя, за которого списываются трудозатраты |
| `Rate` | decimal | Рейты |
| `Comments` | string | Комментарий |

### Вызов и пример использования

**Вызов:**
```
PUT /api/taskexpenses/{id}
```

Где `{id}` – идентификатор трудозатрат.

#### Пример

Изменим трудозатраты с `Id = 666` — меняем количество минут и комментарий:

```
PUT /api/taskexpenses/666
```

```json
{
  "Minutes": 45,
  "Comments": "Изменение минут в трудозатрате 666"
}
```

### Проверка доступа

1. Текущий пользователь должен быть назначен на сервис, в котором находится заявка `TaskId`.
2. У пользователя в типе заявки для `TaskId` в основных полях заявки у поля «Трудозатраты» должно быть выставлено полномочие **«Изменение»**.
3. Если текущий пользователь пытается изменить трудозатраты другого `UserId`, то у него должны быть права **«Редактировать чужие трудозатраты»**.
4. Если в запросе передается новое значение `UserId`, этот пользователь должен быть исполнителем на заявке, и у текущего пользователя должно быть право **«Редактировать чужие трудозатраты»**.
5. Если в запросе передается новое значение `Rate` и у текущего пользователя нет прав **«Редактировать стоимость работ»**, это значение будет игнорироваться.

## Метод GET. Получение возможных исполнителей для списания трудозатрат

Ресурс: `/api/expensesexecutors`

### Поля для получения

| Название | Тип | Описание |
| :--- | :--- | :--- |
| `Id` | Int | Идентификатор пользователя |
| `Name` | String | Имя пользователя |
| `Rate` | decimal | Рейт пользователя в соответствии с сервисной ролью |

### Вызов

**Вызов:**
```
GET /api/expensesexecutors?taskid={taskid}&searchstring={searchstring}
```

| Параметр | Обязательность | Описание |
| :--- | :---: | :--- |
| `taskid` | Да | Номер заявки, для которой пытаемся добавить трудозатраты |
| `searchstring` | Нет | Строка поиска в имени пользователя |

#### Поведение

- Если `searchstring` не передается или передается пустая строка — показывает исполнителей заявки (также из группы исполнителей на заявке) и текущего пользователя.
- Если `searchstring` задан — показывает пользователей с ролью типа «Исполнитель» на сервисе, у которых есть вхождение заданной строки в имя пользователя.

#### Пример вывода в JSON

```json
[
  { "Id": 45, "Name": "тестовый исполнитель 2", "Rate": 5.00 },
  { "Id": 43, "Name": "Администратор", "Rate": 0.00 }
]
```

### Проверка доступа

1. Текущему пользователю должна быть доступна на просмотр заявка `taskid`.
2. Поиск по условию `searchstring` осуществляется в соответствии с настройкой доступа «Не ограничивать просмотр/редактирование пользователей/подразделений своим подразделением и нижестоящими».
