# Категории

Ресурс: `/api/category`

Категории используются для классификации заявок по различным признакам. Они могут иметь иерархическую структуру (родитель-потомок), что позволяет создавать детализированные системы классификации.

## Метод GET. Получение списка категорий и определенной категории

### Поля категории для получения

| Название | Тип | Используется в search | Описание |
| :--- | :--- | :---: | :--- |
| `Id` | Int | | Идентификатор категории |
| `Name` | String | + | Название |
| `Description` | String | | Описание |
| `ParentId` | Int | | Идентификатор родительской категории |
| `Path` | String | | Путь. Идентификаторы категорий с корневой по текущую через разделитель "\|" |
| `BreadCrumbs` | String | | Путь. Названия категорий с корневой по текущую через разделитель "/" |

### Получение списка категорий

**Вызов:**
```
GET /api/category?serviceid={serviceid}&search={searchstring}&fields={fieldlist}&pagesize={value}&page={value}&isTaskHintMode=true
```

Список категорий возвращается в отсортированном виде (по уровню иерархии).

#### Возможные параметры

| Параметр | Обязательность | Описание |
| :--- | :---: | :--- |
| `serviceid` | Нет | Идентификатор сервиса. Позволяет получить категории, привязанные к сервису. Для назначения на заявку нужно передать идентификатор сервиса этой заявки |
| `fields` | Нет | Перечень полей ресурса «Категории» для вывода |
| `search` | Нет | Строка поиска. Ищет в поле «Название» |
| `page`, `pagesize` | Нет | Параметры отображения таблицы |
| `isTaskHintMode` | Нет | Включает режим показа в виде подсказки выбора, как на карточке заявки. Работает только со значением `serviceid > 0` |

#### Примеры

**Пример 1: Список полных названий (с иерархией) всех категорий**
```
GET /api/category?fields=Name
```

**Пример 2: Идентификаторы и названия категорий сервиса 31**
```
GET /api/category?serviceid=31&fields=Id,Name
```

**Пример 3: Поиск категорий по подстроке "Test"**
```
GET /api/category?search=Test
```

### Получение определенной категории

**Вызов:**
```
GET /api/category/{categoryId}
```

Где `{categoryId}` – идентификатор категории.

#### Пример вывода

```
GET /api/category/22
```

```xml
<CategoryView>
  <BreadCrumbs>Аппаратная ошибка</BreadCrumbs>
  <Description/>
  <Id>22</Id>
  <Name>Аппаратная ошибка</Name>
  <ParentId i:nil="true"/>
  <Path>22</Path>
</CategoryView>
```
